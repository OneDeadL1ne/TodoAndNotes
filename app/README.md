# Todos And Notes

Веб-приложение для структурирования мыслей на Python с использованием Flask и SQLite.

В приложении есть две основные страницы:

- `Todo` - список дел.
- `Notes` - заметки и бессистемные идеи.

## Возможности

- добавление заметки;
- просмотр всех заметок;
- удаление заметки;
- добавление задачи;
- просмотр всех задач;
- отметка задачи выполненной;
- удаление задачи;
- статистика по заметкам и задачам.

## Запуск

### Вариант 1. Запуск через exe

Откройте файл:

```text
dist/TodosAndNotes.exe
```

После запуска приложение автоматически откроет браузер. Если браузер не открылся, перейдите вручную:

```text
http://127.0.0.1:5000
```

Для остановки приложения закройте окно `TodosAndNotes.exe`.

### Вариант 2. Запуск через Python

1. Установите Python 3.10 или новее.
2. Установите зависимости:

```powershell
py -3 -m pip install -r requirements.txt
```

3. Запустите приложение:

```powershell
py -3 app.py
```

4. Откройте в браузере:

```text
http://127.0.0.1:5000
```

Для остановки сервера нажмите `Ctrl+C`.

## Сборка exe

Exe-файл уже собран в папке `dist`. Чтобы собрать его заново, выполните:

```powershell
py -3 -m pip install pyinstaller
py -3 -m PyInstaller --onefile --name TodosAndNotes --add-data "templates;templates" --add-data "static;static" app.py
```

Готовый файл появится здесь:

```text
dist/TodosAndNotes.exe
```

## Структура проекта

- `app.py` - Flask-приложение, маршруты и работа с SQLite.
- `database.db` - база данных SQLite, создается автоматически при первом запуске.
- `dist/TodosAndNotes.exe` - готовый exe-файл для запуска приложения.
- `requirements.txt` - зависимость Flask.
- `templates/base.html` - общий HTML-шаблон.
- `templates/index.html` - главная страница со статистикой.
- `templates/notes.html` - страница Notes.
- `templates/todos.html` - страница Todo.
- `static/style.css` - стили интерфейса.

## База данных

Используются 3 таблицы:

- `users` - пользователь приложения;
- `notes` - заметки и идеи;
- `todos` - задачи.

Даже без авторизации таблица `users` нужна для описания предметной области. При запуске создается один пользователь по умолчанию.
